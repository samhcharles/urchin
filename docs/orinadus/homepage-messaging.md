# Orinadus Founder Mode Consolidation

This is the unified scope baseline for brand, product, execution, and presentation.

## 1. Core Architecture (No Jargon)

### Orinadus (orinadus.com)
The macro ecosystem.
It is the destination and overarching platform for AI workflow evaluation and informatics.
It observes, evaluates, and manages.

### Urchin by Orinadus
The universal substrate.
The memory bus that bridges local, cloud, web, CLI, and diverse AI model workflows into one unified context core.
It fixes memory fragmentation.

### Brain Hatching
The onramp.
Users spin up a unified memory core via web (`orinadus.com/secondbrain`) or CLI.
It exports cleanly to local Markdown vaults (for example Obsidian) or remains in the Orinadus cloud ecosystem for dashboard management.

## 2. Immediate Action Plan (Stop the Scatter)

### Step A: Local Monorepo (WSL + Windows 11)

Goal: bring isolated agents, bots, and backend tooling into one centralized foundation.

1. Consolidate
- Move code into one unified monorepo structure in Ubuntu WSL.

2. Develop
- Keep coding inside WSL (Zed workflow) for speed and clean environment boundaries.

3. Bridge
- Install and use `wslu` for robust Linux-to-Windows browser testing loops.
- Route WSL dev servers directly into Windows browser workflows.

### Step B: Minimal Cloud Gateway (Hostinger KVM2)

Goal: make KVM2 the routing node, not a heavy public-hosting layer.

1. Lightweight API gateway
- Deploy a minimal API on VPS as the first Urchin memory-core endpoint.

2. Dogfooding loop
- Connect current terminal/CLI agents to this endpoint.
- Push and pull context continuously to validate live Urchin bridging under real usage.
- Keep traffic private and controlled to respect KVM2 resource limits.

## 3. Messaging Guardrails
